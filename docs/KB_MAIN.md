# База Знаний - Главная страница

## Обзор

База знаний служит долговременной когнитивной памятью проекта. В то время как Supabase хранит транзакционные данные, Notion (или аналогичный инструмент) аккумулирует знания: решения, практики, шаблоны, ошибки и уроки. Правильно организованная база знаний позволяет новым участникам проекта быстро входить в контекст и снижает зависимость от отдельных людей, превращая личный опыт в коллективный.

## Оптимальный инструмент

Рекомендуется использовать **Notion** с использованием нескольких связанных баз данных и главной страницы‑портала. Важно не только создать эти базы, но и приучить себя работать через них: фиксировать новые SOP, добавлять инциденты, обновлять Tech & API Docs и промпты для AI‑агентов.

## Структура базы знаний

Главная страница выполняет роль навигационного хаба и краткого executive‑summary проекта. Она содержит:

1. **Краткое текстовое описание системы и её цели**
   - Что автоматизируется
   - На какие рынки нацелен проект
   - Какой базовый стек технологий используется

2. **Блоки‑ссылки на ключевые базы:**
   - [Products Research](#products-research) — исследование товаров и ниш
   - [Suppliers](#suppliers) — реестр поставщиков
   - [Marketplaces & Accounts](#marketplaces--accounts) — список платформ и аккаунтов
   - [SOPs & Playbooks](#sops--playbooks) — стандартные операционные процедуры и тактические сценарии
   - [Tech & API Docs](#tech--api-docs) — техническая документация по интеграциям и инфраструктуре
   - [AI Prompts & Workflows](#ai-prompts--workflows) — библиотека промптов и AI‑цепочек
   - [Errors & Incidents](#errors--incidents) — журнал ошибок и инцидентов

3. **Дополнительная информация:**
   - Общий Roadmap
   - Ссылки на репозиторий dropshipping‑automation
   - Основные каналы коммуникации (Slack/Telegram)
   - Краткий список текущих приоритетов

## Рекомендации по использованию

1. **Регулярное обновление**: Привычка фиксировать новые знания сразу, а не "потом"
2. **Версионирование**: Отмечайте даты изменений и авторов
3. **Связность**: Используйте ссылки между базами для создания связной структуры
4. **Доступ**: Ограничьте доступ только необходимым людям
5. **Резервное копирование**: Регулярно экспортируйте данные

## Интеграция с системой

База знаний интегрируется с системой через:
- **Supabase** — для синхронизации данных о товарах и поставщиках
- **n8n workflows** — для автоматического обновления некоторых разделов
- **AI агенты** — для использования знаний в принятии решений
