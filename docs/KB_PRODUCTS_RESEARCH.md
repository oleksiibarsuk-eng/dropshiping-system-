# База Products Research

## Назначение

Используется как единая точка для сбора, оценки и приоритизации товарных гипотез. В неё попадает всё, что потенциально может продаваться: от конкретных моделей камер до категорий аксессуаров, нишевых товаров и т.д.

Важно фиксировать не только «что это за товар», но и контекст — почему он вообще попал в рассмотрение, какие сигналы спроса или интуитивные гипотезы за этим стоят.

## Структура базы данных

### Минимальные поля

#### Name
- **Тип**: Текст
- **Описание**: Название товара (конкретная модель или описательное название, если модель ещё не выбрана)
- **Пример**: "Sony A7III", "Canon EF 50mm f/1.8", "Fuji X-T4"

#### Category
- **Тип**: Select
- **Описание**: Категория/сегмент
- **Варианты**: cameras, lenses, accessories, collectibles, other
- **Пример**: "cameras"

#### Niche/Use-case
- **Тип**: Текст
- **Описание**: Целевая аудитория и сценарий использования
- **Примеры**: 
  - street‑фотография
  - travel‑контент
  - блогинг
  - коллекционирование
  - профессиональная съёмка

#### Source marketplace
- **Тип**: Select
- **Описание**: Где товар можно купить
- **Варианты**: eBay.de, японские площадки, локальные магазины, другие источники
- **Пример**: "eBay.de"

#### Source link
- **Тип**: URL
- **Описание**: Исходная ссылка на товар или категорию, из которой он берётся
- **Пример**: "https://www.ebay.de/itm/..."

#### Source price + currency
- **Тип**: Number + Select
- **Описание**: Исходная цена и валюта (с фиксацией примерной даты, если цены сильно волатильны)
- **Пример**: 450 EUR (на 2024-01-15)

#### Target marketplace
- **Тип**: Multi-select
- **Описание**: Где планируется продавать
- **Варианты**: eBay US, Shopify, FB Marketplace, другие платформы
- **Пример**: ["eBay US", "Shopify"]

#### Target price
- **Тип**: Number
- **Описание**: Плановая цена продажи, исходя из желаемой маржи и конкурентной среды
- **Пример**: 650

#### Estimated margin % / €
- **Тип**: Number
- **Описание**: Ожидаемая абсолютная и/или процентная маржа
- **Пример**: 30% или €200

#### Demand signals
- **Тип**: Текст
- **Описание**: Сигналы спроса
- **Что фиксировать**:
  - Продажи/месяц по аналогам
  - Количество конкурентов
  - Тренды выдачи
  - Сезонность
- **Пример**: "15-20 продаж/месяц на eBay US, 3 основных конкурента, пик спроса весной"

#### Notes
- **Тип**: Текст (длинный)
- **Описание**: Специфика товара, потенциальные риски
- **Что фиксировать**:
  - Возвраты
  - Ломкость
  - Гарантия
  - Риски подделок
- **Пример**: "Высокий риск возврата из-за сложности товара, требует тщательной проверки перед отправкой"

#### Status
- **Тип**: Select
- **Описание**: Статус воронки исследования
- **Варианты**: 
  - idea — только идея
  - testing — тестируется
  - active — активен в продаже
  - rejected — отклонён
- **Пример**: "idea"

### Дополнительные поля для приоритизации

#### Impact
- **Тип**: Number (1-5)
- **Описание**: Ожидаемое влияние на бизнес
- **Пример**: 4

#### Confidence
- **Тип**: Number (1-5)
- **Описание**: Уверенность в успехе
- **Пример**: 3

#### Effort
- **Тип**: Number (1-5)
- **Описание**: Сложность реализации
- **Пример**: 2

#### Priority Score
- **Тип**: Formula
- **Описание**: Агрегированный приоритетный скор
- **Формула**: (Impact × Confidence) / Effort
- **Пример**: (4 × 3) / 2 = 6

## Примеры записей

### Пример 1: Конкретная модель камеры

```
Name: Sony A7III
Category: cameras
Niche/Use-case: профессиональная съёмка, travel‑контент
Source marketplace: eBay.de
Source link: https://www.ebay.de/itm/sony-a7iii
Source price: 1200 EUR
Target marketplace: ["eBay US", "Shopify"]
Target price: 1800 USD
Estimated margin: 35% (€420)
Demand signals: 25-30 продаж/месяц на eBay US, стабильный спрос
Status: active
Priority Score: 8.5
```

### Пример 2: Категория аксессуаров

```
Name: Camera Straps (Leather)
Category: accessories
Niche/Use-case: блогинг, street‑фотография
Source marketplace: локальные магазины
Source link: [ссылка на категорию]
Source price: 15-25 EUR
Target marketplace: ["Shopify"]
Target price: 45-65 USD
Estimated margin: 40% (€18-25)
Demand signals: сезонный спрос, пик летом
Status: testing
Priority Score: 6.2
```

## Рекомендации по использованию

1. **Регулярное обновление**: Обновляйте цены и статусы регулярно
2. **Валидация**: Проверяйте актуальность ссылок и цен
3. **Приоритизация**: Используйте Priority Score для выбора товаров для запуска
4. **Связь с другими базами**: Связывайте с базой Suppliers и Marketplaces

## Интеграция с системой

База Products Research интегрируется с:
- **Supabase** — таблица products синхронизируется с этой базой
- **AI агенты** — используются для анализа и приоритизации товаров
- **n8n workflows** — для автоматического обновления цен и статусов
