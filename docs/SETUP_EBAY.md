# Настройка eBay

## Обзор

eBay выполняет роль основного маркетплейса для оперативного тестирования спроса, проверки ценовых гипотез и генерации оборота. В данной архитектуре предполагается разделение ролей: eBay.com — фронт продаж для рынка США, eBay.de — преимущественно источник закупок и локальный аккаунт для EU‑контекста.

Такое логическое разделение позволяет чётко разводить каналы закупки и каналы сбыта, уменьшая вероятность конфликтов интересов и несостыковок в политике площадок.

## ⏱️ Оценка трудоёмкости

2–3 часа активной работы + до 48 часов на прохождение верификаций со стороны eBay. При наличии предварительно подготовленных документов и понимания требований площадки этот этап проходит значительно быстрее.

## Шаги настройки

### 1. Регистрация аккаунтов

#### eBay.com (для продаж на американском рынке)

1. Зарегистрируйтесь на [eBay.com](https://www.ebay.com)
2. Выполните апгрейд до **Business Account** — даёт доступ к расширенным инструментам продавца и снижает некоторые ограничения для новых аккаунтов

#### eBay.de (для закупок и EU‑контекста)

1. Зарегистрируйтесь на [eBay.de](https://www.ebay.de)
2. Выполните апгрейд до **Business Account**

### 2. Верификация аккаунтов

Для обоих аккаунтов выполните:

1. **Верификация личности**:
   - Загрузите скан паспорта или ID документа
   - Подтвердите физический адрес через письмо с кодом или загрузку документа

2. **Привязка банковской карты**:
   - Привяжите банковскую карту для оплаты комиссий платформы

### 3. Создание Developer Account

1. Перейдите на [developer.ebay.com](https://developer.ebay.com)
2. В разделе **My Account** выберите **Create Keyset**
3. Этот шаг критически важен для программного доступа к eBay через API

### 4. Получение Production tokens

Получите следующие токены:

- **Application ID (Client ID)** — используется для идентификации вашего приложения
- **Developer ID** — привязан к вашему аккаунту разработчика
- **Certificate ID (Client Secret)** — для безопасной аутентификации
- **User Token** — полученный через OAuth процесс для доступа к вашему продающему аккаунту

### 5. Настройка Business Policies

Настройте Business Policies, что упростит создание листингов в будущем:

#### Payment Policy
- Установите требование **Immediate payment required** — чтобы избежать неоплаченных заказов

#### Shipping Policy
- **Standard delivery** (7-15 дней) — экономичный вариант
- **Expedited delivery** (3-7 дней) — более быстрая доставка за дополнительную плату

#### Returns Policy
- Предлагайте **30 дней на возврат** с гарантией возврата денег (money back)
- Это повышает доверие покупателей и улучшает ранжирование листингов

## Метрики для отслеживания

Заранее определите, какие метрики вы будете отслеживать на eBay:

- Объёмы продаж по категориям
- Конверсия по типам листингов
- Влияние Business Policies на скорость продаж

Это позволит в будущем подключать аналитику и AI‑агентов не хаотично, а под уже сформулированные вопросы.
