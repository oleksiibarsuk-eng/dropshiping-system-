# Настройка Shopify

## Обзор

Shopify формирует базовый витринный слой системы (D2C‑канал), через который можно: (а) тестировать гипотезы по воронкам продаж, (б) дублировать наиболее маржинальные позиции с маркетплейсов, (в) строить независимый брендовый канал, менее подверженный рискам блокировок со стороны площадок.

Shopify здесь важен не только как «ещё одна площадка», но как опорный элемент экосистемы: через него можно строить собственные лендинги, подключать дополнительные платёжные решения, собирать e‑mail/retargeting‑аудитории и экспериментировать с позиционированием товара вне жёстких рамок маркетплейсов.

## ⏱️ Оценка трудоёмкости

~30–45 минут при наличии базового опыта работы с SaaS‑платформами. При первом знакомстве со средой может потребоваться больше времени на навигацию по интерфейсу и изучение структуры настроек.

## Шаги настройки

### 1. Регистрация и выбор тарифа

1. Перейдите на [shopify.com](https://www.shopify.com)
2. Создайте новый аккаунт
3. Выберите подходящий план:
   - **Basic план** ($39/месяц) — содержит все необходимые функции для старта
   - **Пробный период** (14 дней) — для тестирования системы

### 2. Заполнение деталей магазина

1. **Название магазина** — будет частью URL вашего магазина (например, `yourstore.myshopify.com`)
2. **Физический адрес** — для юридической информации
3. **Валюта** — рекомендуется USD для американского рынка или EUR для европейского

### 3. Создание Admin API Access Token

1. Перейдите в **Settings** → **Apps and sales channels**
2. Выберите **Develop apps** → **Create app**
3. Настройте **Admin API scope**, выбрав необходимые права:
   - `read_products` — чтение информации о товарах
   - `write_products` — создание и изменение товаров
   - `read_orders` — просмотр заказов
   - `write_orders` — обновление статусов заказов
   - `read_inventory` — чтение информации об остатках
   - `write_inventory` — обновление остатков
4. Сохраните полученные **API Key** и **Admin Access Token**

### 4. Дополнительные настройки

1. **Тема магазина** — выберите подходящую тему и зафиксируйте её название в документации
2. **Плагины** — установите необходимые приложения и зафиксируйте список в Tech & API Docs
3. **Ограничения темы** — отметьте любые ограничения выбранной темы

## Важные заметки

Практически полезно на этом этапе сразу зафиксировать в документации (например, в Tech & API Docs) краткую заметку:
- Какой тариф выбран
- Какие плагины установлены
- Какие ограничения есть у выбранной темы

Это избавит от необходимости «вспоминать историю» через несколько месяцев, когда инфраструктура станет сложнее.

## Сохранение данных для интеграции

Сохраните следующие данные в хранилище секретов (см. [SECRETS_MANAGEMENT.md](SECRETS_MANAGEMENT.md)):

- Store URL
- Admin API Key
- Admin Access Token
- Store ID (при необходимости)
- Подключённые домены (при наличии)
- Список основных приложений
