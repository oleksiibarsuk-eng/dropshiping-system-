# Правило работы с кодом и отчетностью

## Обязательное правило для всех изменений кода

После внесения любых изменений в кодовую базу я **ДОЛЖЕН**:

1. **Создать отчет о проделанной работе** в формате markdown
   - Расположение: `reports/{ГГГГ-ММ-ДД}/{ГГГГ-ММ-ДД_ЧЧ-ММ}_{описание_работы}.md`
   - Пример: `reports/2025-11-16/2025-11-16_15-55_dropshipping_system_setup.md`

2. **Обновить Memory Bank** для сохранения знаний о проекте
   - Обновить соответствующие файлы в `docs/rules/memory-bank/`
   - Особенно важно обновлять `context.md` и `architecture.md`

3. **Зафиксировать изменения в git** (если применимо)
   - Сделать коммит с описанием изменений
   - Соответствовать формату коммитов из CLAUDE.md

## Формат отчета о проделанной работе

```markdown
# Отчет: [Название проделанной работы]

## Обзор
[Краткое описание выполненной работы]

## Выполненные изменения
- ✅ [Изменение 1]
- ✅ [Изменение 2]
- ✅ [Изменение 3]

## Созданные/измененные файлы
### Новые файлы
1. `путь/к/файлу1.py` - Описание файла
2. `путь/к/файлу2.py` - Описание файла

### Измененные файлы
1. `путь/к/файлу3.py` - Описание изменений
2. `путь/к/файлу4.py` - Описание изменений

## Результаты тестирования
[Описание результатов тестирования]

## Особенности реализации
[Важные технические детали]

## Преимущества
[Перечисление преимуществ]

## Заключение
[Итоги проделанной работы]
```
